  --task Obstacle-Avoidance-Marl-Direct-v0 \

## 其他补充说明

### 1. VS Code 路径配置

新建 Isaac Lab 项目后，需要在 `.vscode/settings.json` 中添加源码路径，方便 VS Code 正确识别
Isaac Lab 和当前任务包：

```json
{
    "ROS2.distro": "noetic",
    "python.analysis.extraPaths": [
        "/home/zhangl/Isaaclab20260520/IsaacLab/source/isaaclab",
        "/home/zhangl/Isaaclab20260520/IsaacLab/source/isaaclab_tasks",
        "/home/zhangl/Isaaclab20260520/IsaacLab/source/isaaclab_assets",
        "/home/zhangl/Isaaclab20260520/IsaacLab/source/isaaclab_rl",
        "/home/zhangl/Isaaclab20260520/obstacle_avoidance/isaaclab_obstacle_avoidance_task/source/obstacle_avoidance_task"
    ]
}
```

该配置主要用于代码补全和跳转。Python 实际运行时仍需要进入 `isaaclab` Conda 环境，并完成
editable 安装。

### 2. 自定义特征提取器与模型

- `model.py` 定义自定义特征提取器、Policy 网络和 Value 网络。
- `train.py` 与 `play.py` 创建模型，并通过 `agent_cfg["models"]` 传递给自定义 `Runner`。
- `runner.py` 使用 `self._cfg["models"]` 获取这些模型，再创建 MAPPO/IPPO Agent。
- `skrl_mappo_cfg.yaml` 主要保存训练超参数，不直接指定自定义特征提取器。

关键调用关系：

```text
model.py
  → train.py / play.py 创建模型
  → agent_cfg["models"]
  → runner.py
  → MAPPO / IPPO Agent
```

### 3. 车辆 URDF 与配置

- URDF 描述车辆的 link、joint、碰撞体和网格资源。
- `car_cfg.py` 将 URDF 转换为 Isaac Lab 的 `ArticulationCfg`，并配置关节执行器等仿真参数。
- `obstacle_avoidance_env_cfg.py` 根据车辆数量生成每辆车的 Articulation 和 LiDAR 配置。

因此，URDF 负责描述车辆结构，Python 配置负责把车辆接入 Isaac Lab 环境并设置控制方式。

### 4. 环境主逻辑

`obstacle_avoidance_env.py` 中的 `ObstacleAvoidanceMarlEnv(DirectMARLEnv)` 是环境主执行逻辑。
主要通过实现 Isaac Lab 模板钩子完成整体流程：

```text
_setup_scene          创建场景
_pre_physics_step     处理动作
_apply_action         向车辆关节写入控制指令
_get_observations     构造观测
_get_states           构造 MAPPO 全局状态
_get_rewards          计算奖励
_get_dones            判断终止和超时
_reset_idx            重置指定环境
```

### 5. 场景与 LiDAR

- 墙体、静态障碍物、动态障碍物和目标点均通过环境代码创建与布置。
- `obstacle_avoidance_env_cfg.py` 指定每辆车 LiDAR 可以检测的墙体、障碍物和其他车辆。
- 每辆车使用独立 LiDAR 配置，以排除自身车体并获得对应车辆坐标系下的扫描结果。
- 动态障碍物的位置和速度由 `obstacle_avoidance_env.py` 在每个控制步中更新。
